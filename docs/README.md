# Documentation Index

This folder contains operational documentation for running the design blog.

---

## Documents

### [PROJECT_BRIEF.md](../PROJECT_BRIEF.md)
Strategic context for the blog. What it is, why it exists, who reads it, quality bar, and success criteria. **Read this first.**

### [WRITING_GUIDELINES.md](../WRITING_GUIDELINES.md)
Editorial standards for content. Voice, tone, structure, formatting, and guidance for authors (human and AI).

### [SEED_CONTENT.md](./SEED_CONTENT.md)
Launch planning for initial content. What to publish before launch, timeline, and post-launch cadence.

### [TAG_TAXONOMY.md](./TAG_TAXONOMY.md)
Tag structure and governance. Content types, topics, rules for tagging, and Ghost setup.

### [GHOST_WORKFLOW.md](./GHOST_WORKFLOW.md)
Roles, permissions, and editorial workflow. How to set up Ghost, run content through approval, and handle guest authors.

### [IMAGE_PIPELINE.md](./IMAGE_PIPELINE.md)
Image standards and production. Dimensions, formats, compression, alt text, and Figma template guidance.

### [DECISIONS.md](./DECISIONS.md)
Design and implementation decision log. Why things are the way they are, and when to revisit.

## For AI Agents

If you're an AI agent working on this project:

1. **Start with PROJECT_BRIEF.md** — Understand the strategic context
2. **Check DECISIONS.md** — See what's already been decided and why
3. **Follow WRITING_GUIDELINES.md** — If working on content
4. **Reference IMAGE_PIPELINE.md** — If working with images
5. **Update DECISIONS.md** — When making significant implementation choices
